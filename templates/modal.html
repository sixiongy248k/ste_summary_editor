File Recognition
Filename — must contain sum and at least one digit
e.g. summary_1.txt, sum2.txt, summaries_part3.yaml
Extensions — .txt · .json · .yaml / .yml
Recognized Formats
Numbered — 1. First entry or #2. Second entry
Part-based — lines containing part + :
e.g. Part 1: content here, **part 2:** text
JSON — [{"num":1,"content":"..."}]
YAML — 1: First entry content
Re-import — bracket metadata from previous export
#N. (Act|date:val) content
Rejection Reasons
ⓘ Filename — missing “sum” or digit
ⓘ Extension — unsupported file type
ⓘ Empty — file has no content
ⓘ No structure — no numbered entries or part headers
ⓘ Malformed — corrupt or unparseable data
